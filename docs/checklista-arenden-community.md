

# Checklista – hantering av ärenden, frågor och externa bidrag

**Syfte:** Stöd för hur team hanterar issues, frågor och pull requests i öppna projekt.

## A. Bevaka ärenden

- [ ] Det finns en överenskommen frekvens för genomgång av issues och PR:er (t.ex. veckovis).
- [ ] Nya ärenden kategoriseras (bugg, förbättring, fråga, annat).
- [ ] Nya ärenden får en första återkoppling inom rimlig tid.
- [ ] Ärenden med potentiella säkerhetsimplikationer hanteras enligt sårbarhetsrutiner.

## B. Frågor och kontaktvägar

- [ ] Produkt-/projektrelaterade frågor besvaras av ansvarigt team via issues eller motsvarande.
- [ ] Generella frågor om öppen programvara/digital allmännytta hänvisas till överenskommen funktion (t.ex. OSPO eller community).
- [ ] Allmänna frågor om organisationens verksamhet hänvisas till ordinarie kontaktvägar (t.ex. info/registrator).
- [ ] Dessa kontaktvägar framgår i README eller dokumentationen.

## C. Externa bidrag (pull requests)

- [ ] Externa PR:er välkomnas och granskas systematiskt.
- [ ] Kodkvalitet, stil och testbarhet kontrolleras.
- [ ] Säkerhetspåverkan bedöms (särskilt vid ändringar nära gränssnitt eller behörighetshantering).
- [ ] Licens- och copyrightfrågor för bidraget är godtagbara.
- [ ] Påverkan på projektets inriktning är bedömd.
- [ ] Avslag på bidrag kommuniceras respektfullt och begripligt.
- [ ] Contributor Covenant-riktlinjer eller motsvarande uppförandekod används för respektfullt och inkluderande samarbete (Contributor Covenant guidelines - Establishes a social contract for respectful and inclusive collaboration).

## D. Avslutade/arkiverade projekt

- [ ] Om projektet inte längre förvaltas aktivt är detta tydligt i README.
- [ ] Projektet är markerat som arkiverat eller liknande i plattformen.
- [ ] Öppna issues/PR:er hanteras (stängs eller kommenteras) med motivering.
