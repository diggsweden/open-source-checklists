# Säkerhetspolicy

## Rapportera en sårbarhet

Innehållet i detta repo är dokumentation (checklistor) — inte exekverbar kod. "Sårbarheter" handlar därför främst om felaktiga råd som kan leda till osäkra implementeringar i andra projekt.

Om du upptäcker:

- Felaktiga eller missvisande säkerhetsråd
- Brutna länkar till säkerhetsresurser
- Föråldrade rekommendationer
- Andra problem som kan påverka projektets tillförlitlighet

… så rapportera gärna via:

1. **Privat kanal:** öppna en privat rapport via GitHubs Private Vulnerability Reporting (om aktiverat på repot).
2. **Publik kanal:** öppna ett ärende om problemet inte är känsligt.

## Svarsfönster

Vi strävar efter att bekräfta inkomna rapporter inom **5 arbetsdagar** och ge en preliminär bedömning inom **10 arbetsdagar**.

## Stödfönster

Innehållet i detta repo är levande dokumentation och versioneras inte i traditionell mening. Senaste versionen på `main` är den som gäller.
