# Bidra till checklistorna

Tack för ditt intresse! Det här repot innehåller checklistor som ska underlätta efterlevnad av riktlinjer och policy för öppen programvara.

## Hur bidrar du?

1. **Ärenden (issues):** Öppna ett ärende om du upptäcker fel, oklarheter eller saknade ämnen.
2. **Ändringsförfrågningar (PR/MR):** För konkreta ändringsförslag — skicka en ändringsförfrågan.

## Riktlinjer för bidrag

- Skriv på svenska. Använd: *ärende* (issue), *ändringsförfrågan* (PR/MR), *programfel* (bug), *programfix* (patch). Tekniska termer på engelska är OK när det är etablerat (t.ex. *branch protection*, *commit*).
- Följ stilkonventionen: **SKA** för krav, **BÖR** för rekommendationer. Ange källa när det är möjligt (DIGG-riktlinje, OSPS Baseline-kontroll, Standard for Public Code-kriterium).
- Var konkret. En checkpunkt ska gå att svara ja/nej på.
- Undvik att duplicera krav som redan har en kanonisk hemvist i en annan checklista — länka istället dit.
- Behåll "Källor"-sektionen uppdaterad när du lägger till nytt innehåll.

## Commit-meddelanden

Använd [Conventional Commits](https://www.conventionalcommits.org/), exempelvis:

```
docs(sakerhet): lägg till krav på CVD-tidsfönster
fix(licenser): korrigera REUSE-länk
```

## Code review

Alla PR:er granskas av minst en person utöver författaren innan merge.
